# Responsive Static Site Design

**Date:** 2026-04-10  
**Status:** Approved

## Problem

The static site generated by `site/src/generate.ts` has tables that overflow on mobile screens. The matches page (`partidos.html`) has 5 columns (Local, Resultado, Visitante, Fecha, Fase) which is unreadable on typical mobile widths (360–390px). The ranking and stats pages have 3-column tables that are borderline but benefit from improvement.

## Solution

Use the CSS `data-label` technique to transform tables into stacked cards on mobile (≤480px). On desktop the tables remain unchanged.

**Approach chosen:** CSS-only with `@media` + `display: block` on `<td>` (Option A)

- No JavaScript required
- Semantically correct HTML
- Minimal changes to generation logic
- Standard, well-supported technique

## Design

### CSS Changes (in `layout()` function)

Add a `@media (max-width: 480px)` block that:
- Hides `<thead>`
- Sets `table`, `tbody`, `tr`, `td` to `display: block`
- Styles each `<tr>` as a card (border, border-radius, margin, padding)
- Uses `td::before { content: attr(data-label) }` to show column labels inline

### HTML Changes (in generate functions)

Add `data-label` attributes to every `<td>` in each page:

| Page | Columns |
|------|---------|
| `index.html` (ranking) | `#`, `Participante`, `Puntos` |
| `partidos.html` | `Local`, `Resultado`, `Visitante`, `Fecha`, `Fase` |
| `stats.html` | `Resultado`, `Cantidad`, `%` |

### Files Changed

- `site/src/generate.ts` — only file modified:
  1. `layout()`: add responsive CSS block
  2. `generateIndex()`: add `data-label` to ranking `<td>` elements
  3. `generatePartidos()`: add `data-label` to match `<td>` elements
  4. `generateStats()`: add `data-label` to stats `<td>` elements

### Out of Scope

- No changes to Supabase queries
- No changes to GitHub Actions workflow
- No changes to data model or scoring logic
- No additional pages or navigation changes
