# Shutdown de infraestructura productiva — oraculobot

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Dar de baja toda la infraestructura productiva de oraculobot de forma segura, preservando los datos en un dump final antes de eliminar Supabase.

**Architecture:** Outside-in — primero se cortan los triggers externos (crons, webhook), luego se exportan los datos, luego se eliminan los servicios, finalmente se archiva el repo. Este orden evita llamadas huérfanas y garantiza que los datos estén seguros antes de borrar Supabase.

**Tech Stack:** Bash/curl (webhook), GitHub Actions (workflows), TypeScript/tsx (backup script), Wrangler CLI (Cloudflare), gh CLI (GitHub), Supabase dashboard (eliminación manual).

## Global Constraints

- No eliminar Supabase antes de completar la exportación de datos (Tarea 2).
- No archivar el repo antes de completar la eliminación de servicios (Tarea 3).
- Todos los secretos necesarios están en `worker/.dev.vars` (gitignored).

## Tarea 1: Cortar triggers externos

**Archivos:**
- Eliminar: `.github/workflows/check-results.yml`
- Eliminar: `.github/workflows/update-squads.yml`
- Eliminar: `.github/workflows/build-site.yml`

**Interfaces:**
- Produce: repo sin workflows activos, bot de Telegram sin webhook

- [ ] **Paso 1: Eliminar el webhook de Telegram**

  Obtén `BOT_TOKEN` de `worker/.dev.vars`. Luego:

  ```bash
  BOT_TOKEN=<tu_token_aquí>
  curl "https://api.telegram.org/bot${BOT_TOKEN}/deleteWebhook"
  ```

  Respuesta esperada:
  ```json
  {"ok":true,"result":true,"description":"Webhook was deleted"}
  ```

- [ ] **Paso 2: Verificar que el webhook fue eliminado**

  ```bash
  curl "https://api.telegram.org/bot${BOT_TOKEN}/getWebhookInfo"
  ```

  Respuesta esperada: `"url":""` (campo url vacío).

- [ ] **Paso 3: Eliminar los 3 workflows**

  ```bash
  rm .github/workflows/check-results.yml
  rm .github/workflows/update-squads.yml
  rm .github/workflows/build-site.yml
  ```

- [ ] **Paso 4: Commit y push**

  ```bash
  git add .github/workflows/check-results.yml \
          .github/workflows/update-squads.yml \
          .github/workflows/build-site.yml
  git commit -m "chore: eliminar workflows tras fin del Mundial"
  git push
  ```

  Nota: el push dispara `build-site.yml` una última vez — esto es inofensivo porque Supabase todavía existe en ese momento.

## Tarea 2: Ampliar backup.ts y exportar todos los datos

**Archivos:**
- Modificar: `WorldCup2026/backup.ts` — línea 13, constante `TABLES`

**Interfaces:**
- Produce: directorio `backup-output/YYYY-MM-DD/` con un `.json` por tabla

- [ ] **Paso 1: Editar `WorldCup2026/backup.ts` — ampliar TABLES**

  Reemplazar la línea 13:
  ```typescript
  const TABLES = ['users', 'predictions', 'leagues', 'invite_codes'];
  ```
  con:
  ```typescript
  const TABLES = [
    // Tablas de usuario
    'users', 'predictions', 'leagues', 'invite_codes',
    // Tablas operacionales
    'matches', 'knockout_bracket', 'proposed_results', 'question_logs',
    // Tablas WC históricas
    'wc_matches', 'wc_goals', 'wc_referees', 'wc_referee_appearances',
    'wc_bookings', 'wc_substitutions', 'wc_player_appearances',
    'wc_penalty_kicks', 'wc_group_standings', 'wc_award_winners',
    'wc_standings_2026', 'wc_squads_2026', 'wc_coaches_2026',
    'wc_teams', 'wc_stadiums',
  ];
  ```

- [ ] **Paso 2: Correr el script de backup**

  Obtén `SUPABASE_URL` y `SUPABASE_SERVICE_KEY` de `worker/.dev.vars`. Luego desde la raíz del repo:

  ```bash
  SUPABASE_URL=<url> SUPABASE_SERVICE_KEY=<key> npx tsx WorldCup2026/backup.ts
  ```

  Salida esperada (una línea por tabla, ~19 tablas en total):
  ```
  Output directory: /ruta/backup-output/2026-07-22
  Fetching users...
    → N rows written to backup-output/2026-07-22/users.json
  Fetching predictions...
    → N rows written to backup-output/2026-07-22/predictions.json
  ...
  Backup complete.
  ```

- [ ] **Paso 3: Verificar que los archivos existen**

  ```bash
  ls backup-output/2026-07-22/ | wc -l
  ```

  Debe mostrar `23` (una por tabla).

- [ ] **Paso 4: Push al repo de backup**

  ```bash
  git clone "https://github.com/chardila/oraculobot-backup.git" /tmp/oraculobot-backup
  cp -r backup-output/2026-07-22 /tmp/oraculobot-backup/
  cd /tmp/oraculobot-backup
  git add .
  git commit -m "backup: final dump 2026-07-22 (shutdown)"
  git push
  ```

  Si no tienes el PAT configurado, usa:
  ```bash
  git clone "https://x-access-token:${BACKUP_REPO_PAT}@github.com/chardila/oraculobot-backup.git" /tmp/oraculobot-backup
  ```

- [ ] **Paso 5: Commit del backup.ts ampliado al repo principal**

  ```bash
  cd /ruta/al/repo/oraculobot
  git add WorldCup2026/backup.ts
  git commit -m "chore: ampliar backup.ts con todas las tablas para dump final"
  ```

## Tarea 3: Eliminar servicios

**Interfaces:**
- Consume: dump de datos completo en `chardila/oraculobot-backup` (Tarea 2 completada)
- Produce: Worker, Supabase y GitHub Pages eliminados

- [ ] **Paso 1: Eliminar el Cloudflare Worker**

  ```bash
  cd worker
  npx wrangler delete
  ```

  Wrangler pedirá confirmación del nombre del worker (`oraculobot-worker`). Escríbelo y presiona Enter.

  Salida esperada:
  ```
  Successfully deleted oraculobot-worker
  ```

- [ ] **Paso 2: Eliminar el proyecto de Supabase**

  1. Ir a [supabase.com/dashboard](https://supabase.com/dashboard)
  2. Seleccionar el proyecto `oraculobot`
  3. Ir a **Settings → General**
  4. Hacer clic en **Delete project**
  5. Escribir el nombre del proyecto para confirmar

  Nota: esta acción es irreversible. Verificar que `chardila/oraculobot-backup` tiene el dump completo antes de proceder.

- [ ] **Paso 3: Deshabilitar GitHub Pages**

  1. Ir al repo `chardila/oraculobot` en GitHub
  2. **Settings → Pages**
  3. En **Build and deployment → Source**, cambiar a **None**
  4. Guardar
